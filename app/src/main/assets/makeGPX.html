Make GPX file

MTKutility has been upgraded to provide GPX, KML, and CSV translation of the binary data from your GPS logger.  You do not need to connect to your GPS logger to use this.  The file browser defaults to the MTKutility/bin location but you can navigate to other locations.  You should be able to read a file from any user accessible storage location on your phone including your SD card.  Your phone's security will determine what you can do.  The file browser has been designed to let you navigate folders by clicking folder names.  Files are selected by clicking the icon at the left of the file name.  Files that do not have sel in the graphic cannot be selected.

The GPX file is created in MTKutility/gpx.  Provided your loggger is set to record these values, the GPX file can show: latitude - longitude <trkpt lat="0.0" lon="0.0">, elevation <ele>, UTS time <time>, course <course>, speed <speed>, point name (generated value) <name>, reason for recording <type>, type of satellite fix <fix>, number of satellites in use <sat>, Horizontal Dilution Of Precision <hdop>, Positional Dilution Of Precision <pdop>, and Vertical Dilution Of Precision <vdop>.  Note: track point records will not be produced if UTS Date/Time recording is turned off.

The KML file is created in MTKutility/kml.  I adopted the BT747 file format and  xml tags as this appeared to provide the most complete set of options for display in Google Earth.

The CSV file is created in MTKutility/csv.  A spreadsheet with your logger data is the easiest way to discover how various settings affect your track point recording.  I discovered that speed overrides time on my logger. A track point is recorded every half a second when my speed exceeds the set speed.  Note for Excel users: you have to set the Time column cell format to custom hh:mm:ss.0 to get the correct time display.

<fix> values:
nofix - no position fix
SPS - standard position service
DGPS - differential GPS used
PPS - military GPS used
RTK - Real-time kinematic positioning
FRTK - Float Real-time kinematic positioning
est - estimated position
fix values that are outside this list will display as a numeric value

<type> values:
T - time
D - distance
S - speed
B - button (produces a way point)
and any combination of these 4 values
