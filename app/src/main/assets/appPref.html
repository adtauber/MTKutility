app preferences

Click the wrench symbol at the top right to open the preferences screen.  Preferences modify how MTKutility works.  Knowing how to use them will make MTKutility a more positive experience.

The Home and Update Settings screen font sizes are configurable.  They defaults are for a 720x1280 pixel device.  Improve your experience by adjusting the font size.  Click the font you want to adjust and enter a new size. Use your phones back button to leave the Preferences screen.  Then refresh the screen by reselecting it in the navigation menu.

Home screen font changes the size of button text.  Use it to expand or shrink the buttons so that the button row with reConnect fits the full width of your screen. Refresh the Home screen by reselecting it in the navigation menu.  Make sure your are not connected to your GPS device when you change the font. MTKutility will hang it you open app preferences while connected to your logger.

The Update Settings screen has 2 font settings.  Change the Settings screen button font to the largest size that will fit in the buttons without distorting them.  The Settings screen table font affects the text in the table below the buttons.  The column size will not change but row height will and the table will become scrollable if your font is too large.

Please email if you need font size adjustments for any of the other screens.

AGPS memory size - days -- is used in Update AGPS to restrain the number of records read from the EPO file.  Most loggers will only store 7 days, but there are some that handle more.  Leave the setting at 7 if you aren't able to determine your loggers EPO record capacity.

log trace level -- sets the detail written to the execution trace file.  Leave it a normal to keep the log file at the smallest possible size.  I will request setting changes when I get involved in problem resolution with you.

MTK command retries -- sets the the number of repeats tried for some of the more difficult interactions.  This does not apply to all the commands issued by MTKutility.

MTK command response delay -- sets the wait time, in milliseconds, between the send of an MTK command and the read for a response.  This setting has the largest impact on the Home screen Connect.  Increase it in increments of 10 if you experience problems with connect failing or hanging.

EPO load send/receive delay -- sets the wait time, in milliseconds, between the send of the EPO binary packet and the read for a response.  Decreasing this will speed up the AGPS update but it may also result in the update hanging.

download send/receive delay -- sets the wait time, in milliseconds, between the request for data and the read for your logger response. The log download seems to be very timing sensitive.  Please review the Download Log screen help document for problem resolution suggestions.

command retry delay increment -- this value, in milliseconds, is added to the EPO load send/receive delay and download send/receive delay each time the reply response is retried.  Use as an alternative to changing the delay settings when you experience log download or AGPS update hanging failures.

Download Block Size -- sets the size of the packet sent by your MTK logger during the download. While MTK loggers will handle requests for more than 2048 bytes, the largest packet that is returned is 2048 bytes.  I chose to use this as the maximum to simplify app coding logic.  Try decreasing this if you experience problems with the log download.

GPS does not support AGPS -- this setting is there for the GPS loggers without AGPS support that do not behave properly when the test for AGPS is done.

GPS does not support logging -- this setting is there for the GPS receivers without logging support that do not behave properly when the test for logging is done.

disable NMEA for download/AGPS update -- the default state for NMEA sentence sending when the Home screen is not displayed is one GGA sentence every 5 seconds.  This change was made to accommodate GPS devices that function erratically when NMEA sentence output is turned off.  However many loggers support NMEA output being turned off and will process the download and update faster when there isn't NMEA traffic to the Android device.  Explore this by turning this setting on and executing an AGPS update and a larger (5-10k records) log download.  If it works, you can try speeding up these by reducing the download send/receive delay and EPO load send/receive delay.  I have my download send/receive delay set at 50 and EPO load send/receive delay set at 100.
