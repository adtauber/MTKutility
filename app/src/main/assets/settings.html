Update Settings

The settings screen has a cross section of GPS settings - not all possible settings but all that directly affect logging.  It is made up of a set of buttons and an HTML report of your GPS device settings. The button rows match the rows in the HTML table.

The Method button has radio buttons for what happens when the logger's recording when memory is full.  Leave this set at Stop if you want to use MTKutility to download track point data.  I do not have enough information to implement download for Overwrite.

The only other button that doesn't show check boxes is the Frequency button.  The by distance and speed over values in the popup have to be whole numbers.  The by time setting can be in tenths of a second - as fast as every tenth of a second (0.1).  Works well for sky divers.  But a by time setting under 1 second is not saved when you turn the logger off.  The logger reverts to its default value. The logger capacity information is based on a memory size value in device information.  It may not be correct for your logger.  The bitmask at the very bottom is only useful if you are intimately familiar with how the settings are stored.

This screen has 2 size settings: one for the buttons and one for the HTML table.  Adjust them in preferences (wrench at top right and reselect screen after changing) so that everything fits comfortably on your phone's screen.

definitions

In this section I will try to give meaning to the items on the setting screen.  I used the BT747 settings screen documentation and The GPS Dictionary as my reference points.  You can find these documents at BT747 device panel settings and The GPS Dictionary.
Recording
Reason - a code that  defines the reason a track point was recorded
Fix Mode - indicates the kind of fix that was achieved - this includes 'no fix' or invalid, 2D fix, 3D fix and DGPS fix
Fix Only - a directive to the GPS software telling it to only record track points that are derived from a true satellite fix
Method
Overwrite - logger will record continuously by over-writing from the beginning when memory is full (not supported by MTKutility)
Stop - recording stops when memory is full
UTC (Universal time coordinated)
Date/Time - the UTC time for the track point recording -  this always needs to be set if you want a useful log
Mili-second - there are 1000 milliseconds in a second - only useful if you use a recording time that is less than 1 second
Navigation
Latitude - a position's distance north or south of the equator measured by degrees (°)from -90° to +90° or 0° to 90° (N or S)
Longitude - a position's distance east or west of the prime meridian, usually measured in degrees (°) which runs from the North to South Pole through Greenwich, England. Longitude is expressed from -180° to +180° or 0° to 180° (W or E)
Height - height above mean sea level
Speed - the actual speed the GPS unit is moving over the ground
Heading - the direction in which a GPS unit is moving
Distance - the distance from the last track point recorded
Differential
DSTA - the satellite number providing the data
DAGE - how old that data is
Sat Info
NSAT - number of satellites in use
SID - satellite identifier code
Elevation - height above mean sea level
Azimuth - the horizontal direction expressed as the angle from true North.
SNR - signal to noise ratio
DOP  (degree of precision)
PDOP -  position precision - a mix between the HDOP and VDOP value
HDOP - horizontal degree of precision
VDOP - vertical degree of precision
Recording frequency
time - time frequency tor track point recording
distance - only record track points when the distance from the last point recorded exceeds this setting
speed - only record track points when the speed exceeds this setting

The BT747 documentation has an excellent explanation for using these settings.
