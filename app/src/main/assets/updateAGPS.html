Update AGPS

Important: download a new EPO file before you execute the update AGPS function.

This is a 2 step process.  The first step is to select the EPO file by clicking the Select EPO File button.  The default MTKutility/epo directory will open but you can navigate to a different location if your file is not in this folder.

Click the icon to select the file.  Clicking any other part of the row will not select the file.  This unlocks the Update AGPS data button.  MTKutility will limit the AGPS update to the size limit set under the AGPS memory size - days app preference.  You can use a 30 day file for a 7 day update without having to worry about extra processing time for the larger file.  Progress is indicated by the progress circle turning green and the increasing count of sets stored.

You can use the Select EPO File function to delete files from the MTKutility/epo directory. Long press the icon of the file you want to delete.  Select Delete on the popup, or Cancel if you decide not to delete.  You can also Rename.  View is not a useful option here.  Phones do not normally have a binary file viewer. You will not be able to leave MTKutility if the update hangs. You will have to kill the app from your android system.  Make sure you run MTKutility right after you kill it to reset your logger.

dealing with problems that arise
The AGPS update has a timing preference in app preferences - EPO load send/receive delay.  The is wait time, in milliseconds, between the data send and the read for success acknowledgment.  Reducing this will speed up the AGPS update.  However that may also make the update abort or hang.  Increase the delay by 50 milliseconds when this happens.

Very Important: an abort or hang during the update will leave your logger in a non-functioning state and make it impossible to exit MTKutility.  Your logger is put into binary mode when the update starts.  A hang means that your logger is not responding so the code that reverts your logger to NMEA mode is not executed.  The recovery steps are: turn your phone's Bluetooth off, turn off your logger, turn your phone's Bluetooth on, turn on your logger, and run MTKutility again to reset your logger to NMEA mode.
